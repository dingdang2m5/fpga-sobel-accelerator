### 第一步：建立“参考答案” (Python Golden Model)

在写任何代码之前，必须先确定“正确的结果长什么样”。

- **任务**：写一个简单的 Python 脚本。
  1. 读取一张灰度图。
  2. 使用 OpenCV 跑一遍 Sobel 算子，或者手动用 Numpy 写一个 $3 \times 3$ 卷积逻辑。
  3. **关键点**：将输入的像素值导出为 `input_pixels.txt`，将算出来的结果导出为 `golden_output.txt`。

------

### 第二步：核心模块开发 (The "Hardware Logic")

这是项目的灵魂，建议你们讨论决定是用 **SystemVerilog** 硬敲，还是用 **Vitis HLS** 写 C++。

- **任务**：实现 `Line Buffer` -> `Window Generator` -> `Sobel Engine` 的逻辑。
- **核心攻关**：
  - **Line Buffer**：如何用两行存储来产生三行数据。
  - **Pipeline**：确保计算部分是流水线化的，每个周期能算一个结果。

------

### 第三步：写 Testbench 并跑通波形 (Vivado Simulation)

在 Vivado 里创建一个仿真环境，把第二步写的模块放进去“喂数据”。

- **任务**：
  1. 写一个 Testbench，使用 `$readmemh` 读取第一步生成的 `input_pixels.txt`。
  2. 模拟 AXI-Stream 的握手信号（给 `TVALID`，看 `TREADY`）。
  3. **看波形**：检查计算出的第一个像素是在什么时候出来的？数据有没有错位？

------

### 第四步：构建系统拓扑 (Vivado Block Design)

当单个 IP 仿真通过后，把它集成到 Zynq 系统中。

- **任务**：
  1. 在 Vivado 中新建 Block Design。
  2. 放入 **ZYNQ7 Processing System**、**AXI DMA** 和你们自己的 **Sobel IP**。
  3. 点击 **Run Connection Automation** 让 Vivado 自动生成复位和互联矩阵。
- **验证**：进行“系统级仿真”或者直接生成 Bitstream。

------

### 第五步：板级部署与 Python 调用 (PYNQ)

把生成的 `.bit` 和 `.hwh` 文件拷到 PYNQ-Z2 上。

- **任务**：
  1. 在 Jupyter Notebook 里写代码分配 DMA 缓存。
  2. 搬运数据，启动硬件。
  3. **终极时刻**：用 Python 读回结果，对比它和第一步生成的 `golden_output.txt` 是否像素级一致。
